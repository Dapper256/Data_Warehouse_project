#  Data Catalog: Gold Layer (Analytics & Reporting)

## 📌 Overview
The **Gold Layer** represents the business-level data tier, optimized specifically for analytical modeling, business intelligence, and reporting. Data in this layer is structured using a dimensional modeling approach (**Star Schema**), consisting of highly descriptive **Dimension Tables** and a transactional **Fact Table** to power high-performance dashboarding and executive KPIs.

---

##  1. `gold.dim_customers`

**Purpose:** Stores comprehensive customer profile details enriched with demographic, geographic, and socio-economic attributes.

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| **`customer_key`** | `INT` | *Surrogate Key:* Uniquely identifies each customer record in this dimension table (Primary Key). |
| `customer_id` | `INT` | Unique numerical business identifier assigned to each customer from the source system. |
| `customer_number` | `NVARCHAR(50)` | Alphanumeric business identifier used for external tracking and reference. |
| `first_name` | `NVARCHAR(50)` | The customer's first name. |
| `last_name` | `NVARCHAR(50)` | The customer's last name or family name. |
| `country` | `NVARCHAR(50)` | The country of residence for the customer (e.g., 'Australia'). |
| `marital_status` | `NVARCHAR(50)` | The marital status of the customer (e.g., 'Married', 'Single'). |
| `gender` | `NVARCHAR(50)` | The gender of the customer (e.g., 'Male', 'Female', 'n/a'). |
| `birthdate` | `DATE` | The date of birth of the customer, formatted as `YYYY-MM-DD`. |
| `create_date` | `DATE` | The date when the customer record was originally created in the system. |

##  2. `gold.dim_products`

**Purpose:** Provides a centralized, detailed repository of product attributes, pricing, and structural categorizations.

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| **`product_key`** | `INT` | *Surrogate Key:* Uniquely identifies each product record in this dimension table (Primary Key). |
| `product_id` | `INT` | Unique internal tracking identifier assigned to the product. |
| `product_number` | `NVARCHAR(50)` | Structured alphanumeric code representing the product for inventory control. |
| `product_name` | `NVARCHAR(50)` | Descriptive name of the product (includes type, color, and size variants). |
| `category_id` | `NVARCHAR(50)` | Unique identifier for the product's high-level category. |
| `category` | `NVARCHAR(50)` | Broad classification group of the product (e.g., 'Bikes', 'Components'). |
| `subcategory` | `NVARCHAR(50)` | Narrower sub-classification within the main category (e.g., 'Mountain Bikes'). |
| `maintenance_required` | `NVARCHAR(50)` | Operational flag indicating if the item requires maintenance updates ('Yes' / 'No'). |
| `cost` | `INT` | The acquisition cost or base production price of the product in whole currency units. |
| `product_line` | `NVARCHAR(50)` | The specific product line or series group (e.g., 'Road', 'Mountain', 'Touring'). |
| `start_date` | `DATE` | The date when the product officially became available for market sale (`YYYY-MM-DD`). |

---

##  3. `gold.fact_sales`

**Purpose:** Stores core quantitative, transactional retail metrics and foreign keys connecting back to the dimensional tables.

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| `order_number` | `NVARCHAR(50)` | Unique alphanumeric identifier for the overarching sales order (e.g., 'SO54496'). |
| **`product_key`** | `INT` | *Foreign Key:* Links the transactional record directly to **`gold.dim_products`**. |
| **`customer_key`** | `INT` | *Foreign Key:* Links the transactional record directly to **`gold.dim_customers`**. |
| `order_date` | `DATE` | The operational date when the sales order was submitted. |
| `shipping_date` | `DATE` | The logistical date when the order was fulfilled and shipped out. |
| `due_date` | `DATE` | The financial deadline date by which payment must be completed. |
| `sales_amount` | `INT` | Total monetary revenue value generated by this specific line-item sale. |
| `quantity` | `INT` | Number of physical units purchased for this specific line item. |
| `price` | `INT` | Unit price of the product at the exact time of transaction. |
